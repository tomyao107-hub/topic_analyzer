# 数字人文分析功能版本安排

## 文档状态

- 状态：已确定，待按版本实施
- 基线版本：v2.0
- 安排日期：2026-07-12
- 实施顺序：v2.1 → v2.2 → v2.3 → v2.4

本文档用于约束词频与词云、情感分析、命名实体识别、词嵌入与语义网络四类数字人文能力的开发顺序、功能边界和验收标准。各版本按照实现难度递增，并保持中文、英文语料彼此独立分析。

## 总体安排

| 版本 | 功能 | 难度 | 核心成果 |
|---|---|---:|---|
| v2.1 | 词频分析与词云 | 低 | 高频词表、文档频率、柱状图、词云及结果导出 |
| v2.2 | 情感分析 | 中 | 文献情感评分、情感分类、证据词和历史维度聚合 |
| v2.3 | 命名实体识别（NER） | 中高 | 人名、地名、机构、官职、时间抽取与上下文定位 |
| v2.4 | 词嵌入与语义网络 | 高 | Word2Vec、相似词、语义网络和跨时期语义演变 |

每个版本应独立完成开发、测试和发布，不将尚未完成的后续版本作为当前版本通过验收的条件。

## v2.1：词频分析与词云

### 研究目标

快速呈现文献的核心关注点和高频主题，帮助研究者把握语料的宏观特征。

### 功能范围

- 基于清洗后的分词结果进行分析，不覆盖原始文本。
- 中文、英文分别统计，禁止混用词表。
- 支持按总词频或文档频率排序。
- 支持设置 Top N、最低词频和最低文档频率。
- 展示词语排名、总词频、文档频率和语料占比。
- 提供高频词柱状图和词云。
- 沿用清洗阶段的停用词、自定义词典及语言配置。
- 支持导出词频明细 CSV 和词云 PNG。

### 建议导出文件

```text
zh/word_frequency.csv
zh/word_cloud.png
en/word_frequency.csv
en/word_cloud.png
```

### 验收标准

- 给定固定 tokens，词频和文档频率计算结果准确。
- 中文和英文结果完全隔离。
- 相同输入和参数产生稳定的排序结果。
- 空语料、过滤后无词语等情况给出明确提示。
- CSV 可独立复核，PNG 可正常保存和打开。

## v2.2：情感分析

### 研究目标

识别文献中的褒贬、正负倾向，为文学人物研究、历史舆论分析和社会心理变化研究提供辅助指标。

### 功能范围

- 首版采用透明、可复现的词典与规则方法。
- 中文、英文使用独立的情感资源和参数。
- 输出文献级情感分数以及正面、中性、负面分类。
- 展示命中的正面词、负面词及局部文本证据。
- 支持导入研究者自定义的正面词典和负面词典。
- 支持按年份、来源、文类或自定义元数据进行聚合。
- 展示情感分布和随时间变化的趋势。
- 记录分析方法、阈值和词典配置，保证结果可复现。

### 建议导出文件

```text
zh/sentiment_documents.csv
zh/sentiment_summary.csv
en/sentiment_documents.csv
en/sentiment_summary.csv
```

### 验收标准

- 文献级结果包含分数、分类、证据词和文献编号。
- 自定义词典能够改变结果，且配置写入会话记录。
- 聚合结果可追溯到原始文献。
- 中英文词典和结果互不污染。
- 界面明确说明情感结果是辅助测量，不将算法分类表述为客观事实。

## v2.3：命名实体识别（NER）

### 研究目标

从非结构化历史文本中提取关键实体，为人物数据库、机构关系研究和空间地理分析提供结构化基础。

### 功能范围

- 识别人名、地名、机构名、官职和时间五类核心实体。
- 采用“本地统计模型 + 历史词典 + 规则”的混合方案。
- 使用规则处理日期、年份、年号等时间表达。
- 支持导入历史人物、历史地名、机构和官职词典。
- 保存实体原文、标准化名称、实体类型、文献编号、字符位置和上下文。
- 展示实体频次、实体—文献关系和原文证据。
- 对别名、异体字和简称保留原始值，不自动进行不可逆合并。
- 支持按实体类型筛选、搜索和导出。

### 建议导出文件

```text
zh/entities.csv
zh/entity_mentions.csv
en/entities.csv
en/entity_mentions.csv
```

### 验收标准

- 五类实体均具有明确、稳定的输出结构。
- 每次实体出现都能定位到文献和原文上下文。
- 重叠实体、重复实体和规则冲突具有确定的处理顺序。
- 自定义词典可扩充实体识别结果。
- 通用模型不可用时提供环境提示，不静默返回空结果。
- 历史实体识别结果允许人工复核，不以自动合并替代研究判断。

## v2.4：词嵌入与语义网络

### 研究目标

计算词语之间的语义相近度，并研究特定词汇在不同时期的语义邻域及语义变化。

### 功能范围

- 复用现有 gensim 依赖训练 Word2Vec 模型。
- 支持设置向量维度、上下文窗口、最低词频、训练轮次、训练方式和随机种子。
- 支持查询目标词的相似词及相似度。
- 构建以目标词为中心的语义网络，并控制节点数和相似度阈值。
- 支持按年份或自定义时期字段切分语料。
- 对不同时间片的词向量空间进行对齐后再比较。
- 展示目标词在各时期的近邻变化、相似度变化和语义漂移分数。
- 对语料量、共同词汇量不足的时间片阻止分析或给出显著警告。
- 保存训练参数和随机种子，支持复现。

### 建议导出文件

```text
zh/embedding_neighbors.csv
zh/semantic_network.graphml
zh/semantic_shift.csv
zh/word_vectors.kv
en/embedding_neighbors.csv
en/semantic_network.graphml
en/semantic_shift.csv
en/word_vectors.kv
```

### 验收标准

- 固定语料、参数和随机种子时结果在允许误差内可复现。
- 相似词查询能正确处理未登录词。
- 网络中的节点、边及权重可以导出和复核。
- 不直接比较未经对齐的独立时间片向量。
- 时间片语料不足时不生成误导性的语义变化结论。
- 语义变化结果能够追溯到时间片、参数和训练语料。

## 统一实施要求

每个版本均按以下步骤执行：

1. 在 `services/` 新增独立分析服务，避免把算法逻辑堆积在桥接层。
2. 在 `backend/bridge.py` 增加任务命令、参数校验和结果序列化。
3. 在 `models/app_state.py` 增加按语言隔离的结果状态。
4. 扩展 `desktop/src/state/workflowStore.ts` 的任务、配置和结果类型。
5. 在 React 桌面界面中新增导航页、参数区、结果区和可视化。
6. 将新结果接入统一导出流程和 `session_config.json`。
7. 增加服务单元测试、桥接测试、双语隔离测试和异常路径测试。
8. 更新 README、版本号和使用说明。

## 界面与流程安排

建议最终导航顺序如下：

```text
总览 → 导入 → 清洗 → 词频 → 情感 → 实体 → 语义 → LDA → STM → 对比 → 导出
```

四类新分析均以“已完成清洗”为前置条件。LDA、STM 和现有对比分析继续保持原有行为，不因新增功能改变其结果格式。


同时应检查：

- 原始 `text` 字段不被覆盖。
- 新增参数写入会话配置，分析结果能够复现。
- 错误提示包含可采取的修复建议。
- 文档中的导出文件名与实际实现一致。

## 研究方法边界

- 词频反映出现次数，不自动等同于主题重要性。
- 情感分类是算法测量结果，需要结合文本证据和历史语境解释。
- 历史实体存在别名、异体字和行政区划变化，自动标准化必须可审查、可撤销。
- 不同时间片独立训练的词向量空间不能直接比较，必须先进行空间对齐。
- 数据量不足时优先提示限制，不输出形式完整但研究意义不足的结论。
